# Novel Review Tool

小说多维度自动评价工具。输入一个小说 txt 文件，通过多阶段 LLM 分析，输出带证据链的结构化评价报告。

## 功能

- **8维度评分**：情节结构、人物塑造、文笔质量、世界观构建、主题深度、对话质量、情感感染力、创新性（每个维度附带 rubric 锚点）
- **缺陷检测**：D01-D08 共8条启发式规则（龙傲天、金手指过度、感情线突兀、节奏塌陷、人设崩塌、设定遗忘、注水严重、伏笔断裂）
- **双视角点评**：文学性视角 + 商业性视角
- **证据链追溯**：每个评分绑定原文引用
- **长篇支持**：>50万字自动切换抽样精读模式，附 coverage/confidence 披露
- **断点续传**：每块独立落盘，中断后可 `--resume` 继续
- **实时进度**：运行时写入 `progress.json`，可随时查看分析进度

## 快速开始

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，填入你的 API 密钥

# 3. 运行
cd src
python -m novel_review --input "../data/你的小说.txt"
```

## CLI 参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--input, -i` | 输入 txt 文件路径（必填） | - |
| `--output, -o` | 输出目录 | `output/` |
| `--resume` | 断点续传，跳过已完成的块 | false |
| `--force` | 强制重新分析所有块 | false |
| `--mode` | `full`=全量精读, `auto`=根据字数自动选择 | auto |

## 处理流水线

```
Phase 0: 文本预处理 → 编码检测 → 章节识别 → 分块清单
    ↓
Phase 1a: 全量轻筛 → 每块生成摘要、节奏分、候选标签
    ↓
Phase 1b: 精读分析 → 关键块深度分析（短篇全量，长篇抽样）
    ↓
Phase 2: 跨块综合 → 节奏曲线、人物弧光、伏笔配对、缺陷检测
    ↓
Phase 3: 双视角评价 → 8维度评分 + 总评 + 改进建议
    ↓
输出: report.md + report.json
```

## 长篇抽样模式（>50万字）

当小说超过50万字时自动启用抽样策略：

1. **全量轻筛**：所有块都做轻量分析（情节摘要、节奏分、状态变化）
2. **关键块精读**：
   - 前3块（开篇）+ 后3块（结尾）
   - 高节奏且有主线推进的块
   - 有关键标签的块（高潮、转折点）
   - 有重大状态变化的块
   - 每10块至少抽1块
3. **confidence 披露**：
   - 情节结构、人物弧光：基于全量轻筛，置信度较高
   - 文笔、主题、情感：仅基于精读块，抽样模式下置信度降为 low
   - 报告中明确标注每个维度的 confidence 级别

**限制**：抽样模式下，精读覆盖率通常在 20-40%，文笔和主题评分仅反映被抽样段落的水平，可能不代表全书整体。使用 `--mode full` 可强制全量精读（耗时和成本成倍增加）。

## 成本参考

以 Gemini API 为例：

| 小说规模 | 分块数 | LLM 调用次数 | 约 Token 消耗 | 预计耗时 |
|----------|--------|-------------|--------------|---------|
| 短篇（<10万字） | ~15块 | ~32次 | ~80K | 5-10分钟 |
| 中篇（10-50万字） | ~60块 | ~125次 | ~375K | 15-30分钟 |
| 长篇抽样（>50万字） | ~200块 | ~250次 | ~600K | 60-90分钟 |
| 长篇全量（>50万字） | ~200块 | ~400次+ | ~1.2M+ | 2-4小时 |

实际耗时和费用取决于所选模型和 API 定价。

## 输出文件

```
output/<书名>/
  report.md        # Markdown 格式评价报告
  report.json      # JSON 结构化数据
  progress.json    # 运行进度（分析完成后保留）

artifacts/<书名>/
  chunks.json              # 分块清单
  phase1_light/            # 轻筛结果（每块一个 JSON）
  phase1_deep/             # 精读结果（每块一个 JSON）
  phase2/
    cross_chunk_analysis.json  # 跨块综合分析
    evidence_index.json        # 证据索引
```

- 子维度列表

## 缺陷检测规则

| 规则 | 名称 | 检测方式 |
|------|------|----------|
| D01 | 龙傲天 | LLM 标注 |
| D02 | 金手指过度 | LLM 标注 |
| D03 | 感情线突兀 | LLM 标注 |
| D04 | 节奏塌陷 | 启发式：连续3块 pacing<=2 且无主线推进 |
| D05 | 人设崩塌 | LLM 标注 |
| D06 | 设定遗忘 | 启发式：设定属性前后矛盾 |
| D07 | 注水严重 | 启发式：连续2块 information_density<=2 |
| D08 | 伏笔断裂 | 启发式：伏笔埋设后>=5块未回收 |

## 已知限制

- 评分受 LLM 模型偏好影响，同一作品换模型可能有 1-2 分波动
- 世界观构建维度在现实题材小说中往往得分偏低（缺乏奇幻/科幻设定要素）
- 缺陷检测中 D01-D03、D05 主要依赖 LLM 判断，无法纯启发式验证
- 证据引用来自 LLM 提取，偶有不精确的情况
- 长篇抽样模式下部分维度置信度有限
